✦ Core Abstraction

Textron models task experience as a multi-layer graph of text nodes. Each task family gets its own network. Nodes are arbitrary text, edges are weighted connections. Through forward propagation (LLM attention scoring → layer filtering → prompt compilation) and backward propagation (feedback → weight updates → node filling), the network converges to produce optimal context for recurring tasks.

The LLM assigns attention scores to each node based on relevance to the current task. The propagate action runs layer-by-layer: score × edge_weight → activation sum → threshold filter → next layer. Activated nodes are compiled into the execution prompt.

✦ Convergence Over Time

Task 1 (fresh network)

All nodes empty. LLM executes task. → calls backward to fill 4 nodes. → no context help yet.

Task 2 (network has content)

LLM routes → forward scores 4 filled nodes. → propagate activates them. → prompt: "useMemo, stale closure, ..." → LLM solves task faster. → backward reinforces edges.

Task 5 (partially converged)

6/10 nodes filled, edges converged. → forward activates high-weight path. → prompt is shorter but more accurate. → user corrections reduced 70%. → LLM fills remaining empty nodes.

Task 20 (converged)

All nodes filled with refined patterns. → edge weights near +1 for successful paths. → negative weights suppress dead ends. → prompt: compact, exact, zero fluff. → network can be distilled into a SKILL.md.

✦ Storage Architecture

Filesystem (no database)

~/.textron/ └── react_hooks_debugging/ ├── hyperparams.json # layers, threshold, lr ├── weights.json # all edge weights ├── layer_0/ │ ├── node_0.html # <content>text</content> │ ├── node_1.html │ └── ... ├── layer_1/ └── ...

Node HTML Format

<meta name="layer" content="0"> <meta name="id" content="node_0"> <link rel="out" href="../layer_1/node_1.html" data-weight="0.54"> <content> useMemo to stabilize object references in deps </content>

HTML nodes + JSON weights = human-readable, git-versionable, browser-inspectable. No predefined node types — a node's role emerges from its layer position, connections, and content.
